一些碎碎念。
这个工程开始于2026/3/15.它现在还只是躺在我的ide里的一个孤零零的，只有几个token，opcode定义的文件。
事实上，作为一个从业游戏行业四年的美术从业者，完全从零开始去开始这样一个“编译器”项目，所面对的阻力和困难是巨大的。事实上我一路上走来一直都很迷茫。
在一开始(2025年)，我只是想用excel做一个简单的跑团数值模拟器，在遇到一些新功能——比如让玩家“自定义”一个技能——时，难以想象的阻力出现了，这逼迫我不得不转向一个表达力更强的工具。

——我选择了vibe coding——
在一开始，一切都顺风顺水，我敲下需求，ai帮我实现，似乎一切都近在眼前，我当时甚至狂妄的想，只要我一个人+AI，我就能搞出所有人都搞不定的项目。
然而现实总是在人最狂妄时同样狂妄的展示自身的力量——第一个项目崩了，不论我如何调整关键词，如何尝试重构，甚至后面要求AI重写整个项目，问题始终都无法解决。
这是当然是理所当然的，因为当时的我甚至连环境配置是什么都不知道，更遑论去了解为什么ai要用nodejs，什么是OOP，什么是设计模式，什么是dod，什么是ecs，语言为什么这么写。
我开始考虑是框架的问题，或许nodejs这门语言的框架问题导致了这一切——当然，现在看来并不是如此。

于是我转向了flutter。
一开始一切都好，在经历了又两三次重新新建文件夹之后，我甚至能脱离ai，自己写一些不错的/不包含和后端交互的纯画面表现的代码出来。
当时我觉得胜利尽在眼前。
然而当复杂逻辑开始介入，软件需求变得越来越复杂时，dart和flutter的局限性开始出现了。
或许是时候考虑是否是自己的问题了。
当时的我消沉了相当长一段时间，我甚至考虑，是否我本身就不适合学代码。万幸的是AI的出现降低了我学习任何事物的门槛，因此重新振作之后，我再次狂妄起来——何不直接用写出dart的语言——也即C系语言，来实现这个项目？

又是一样的顺风顺水，又是一样的AI接管一切，又是一样的多次重构，直到代码行数来到一万行左右，来到我认为最接近胜利的那一刻。
此时的我手握在ENTT上构建出的ECS后端，在一切都由AI所写就的幻觉中，我的引擎甚至跑出了一个可以说得上是惊人的运行速度。
我当时认为自己已经是新时代能掌握Vibe coding，对ai使用最熟练的哪一类弄潮儿了。
直到我看到那一堆我看不懂的代码，我才意识到，我只是在被ai所引导。

帝国又一次崩塌了，或许仅靠AI是无法达成我的目标的。

怀疑不可避免，对编程这件事的厌恶开始上涌，然而在这半年多的vibe coding经历中，我也不可避免的领略到编程这项技能的不同之处，以及它的美。
我开始意识到，事实上每一门语言都是一个曾经的人，在一个曾经的夜晚，一个清晨，一个屏幕面前，面对自己所敲下的一行行复杂代码时所发出的那个疑问。
“能不能再好一点？”

————能不能再好一点？我同样这样问自己。

我开始注意到那些发紫，发蓝的，被ide所高亮的各类语法，开始幻视那些无数个和我一样抓耳挠腮，不知如何深入的人，看到那些和我一样迷茫，不知道前路在何方，只能紧抓这一丝逻辑和实在的人。
代码是冰冷的，但它是坚实的，是属于我的，我的表达，我的思想，是我用那些先贤构造出的语言所讲出的属于我的故事。

我想这正是编程这门艺术的迷人之处——它看起来冰冷，不近人情，然而却能表达出创作者的思想，情感，以及他在无数个深夜中的每个抉择。

我想要完成这个项目，我想要用它来表达我对这个世界的理解，我想要用它来表达我对这个世界的情感，我想要用它来表达我对这个世界的思考，我想让我的灵魂通过这样一门语言，寄宿在电子网络中，像幽灵一样传递，漂流，去和无数个和我一样的人交流。

亲爱的，如果你看到这里，我衷心的感谢你，谢谢你的参与，谢谢你的支持，谢谢你的陪伴。

祝你在这条路上永保热爱，风雨无阻。
