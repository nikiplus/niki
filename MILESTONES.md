## M1 - V0 最小闭环

### 目标
- 构建可运行的最小语言闭环，优先保证语法-语义-执行链路可贯通

### 交付物
- 基础表达式：字面量、变量、二元运算、赋值
- 控制流：if、loop、break、return
- 函数：func + 参数 + 返回
- 类型：int / float / bool / string / void
- 结构：仅 struct（AOS）
- 模块：最小 module / 文件级作用域

### 验收标准
- [ ] 语法可解析并生成 AST
- [ ] 基础语义可通过（最小 checker 规则）
- [ ] 基础样例可编译并执行
- [ ] 表达式解析未完成项已明确标注为“暂定”

### 风险与阻塞
- 表达式分支仍有未落地路径，可能影响 V0 完整性评估

## M2 - V0.5 极简系统层

### 目标
- 在不引入复杂并发语义前提下，完成 system/component/flow/kits 最小可用版本

### 交付物
- component（SOA）：仅“声明 + 读写 API”，不做复杂查询优化
- system：仅支持同步执行，不允许 await
- flow：仅顺序调度 system，不支持并发
- kits：仅“显式依赖清单”，不引入所有权魔法

### 验收标准
- [ ] system-component 读写链路可跑通
- [ ] flow 顺序调度可验证
- [ ] kits 依赖声明可解析且可检查

### 风险与阻塞
- 语义边界若定义不清，后续 V1 并发扩展会返工
